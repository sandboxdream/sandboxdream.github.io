教你如何使用github page和Hexo搭建一个博客
2020-07-09
Word count: 2.1k|Reading time: 7 min

选择托管平台

目前国内主流的托管静态网站的平台有两个：github和gitee。下面将叙述两个平台各自方优缺点

github

github是目前全球都在用的一个开源共享平台，它具有托管静态网页的功能，且由于使用此平台的程序猿众多，你几乎不用担心这个平台遭遇ao3的待遇。
优点 ：
1、不受到中国网络审查
2、上传后自动更新静态页面
缺点：
1、访问速度较慢（加速访问方法放在文末）

gitee

gitee（码云）是CSDN发起的一个更适合中国人使用习惯的开源代码平台。
优点：
1、访问速度快
缺点：
1、接受国内网络审查，可能一不小心你的号和博客上所有东西就没了
2、每次上传完成后都需要登录并更新页面

(以上优缺点是我在之前的使用过程中发现的，如果有补充可以提交评论)

本教程推荐使用github作为托管平台

本地操作

需要的软件

下载并安装nodejs

打开nodejs的官网并选择相应的平台下载

安装选项全部默认就可以了。

按windows+R输入cmd打开命令提示符
输入 node -v 和 npm -v 如果出现版本号则安装成功

下载并安装git

打开git的官方网站并下载

安装选项同样全部默认。

安装完成后在命令提示符中输入 git --verson 验证时候安装成功

添加国内镜像

使用阿里的镜像对下载加速

安装hexo

在合适的地方新建一个文件夹用于存放博客的文件
然后右键，点击Git Bash Here

然后输入

来安装hexo
输入 hexo -v 检查安装

创建本地文件

在刚刚打开的gitbash中输入 hexo init 在这个文件夹创建hexo博客文件
然后输入 npm install 来加载未加载项
这两步的时间耗时可能巨大，请做好心理准备

修改_config.yml文件

这里推荐安装notepad++或vscode对配置文件进行编辑。当然，如果你不想安装也是可以通过windows自带的记事本进行编辑的
在你刚刚创建本地文件的文件夹下找到名为_config.yml的文件，并选择打开方式为notepad++或vscode或记事本

测试本地服务器

以上两步弄好后输入 hexo g 编译页面，然后输入 hexo s 创建本地预览服务器，在浏览器地址栏输入 localhost:4000 查看(或点击这里)

至此。如果你本地预览正常，那么恭喜你，你已经成功1/3了。

github上的操作

注册账号

首先，你需要注册一个github账号。
点击这里打开github官网。
界面都是英文的，但是不需要担心，你只需要照着做就可以了。如果你使用chrome，谷歌自带的翻译会自动启用。你可以照着机翻来做。
加载会有一点慢，或者直接提示找不到服务器之类的，请百度/google/bing一下github加速方法或等待我的另一篇加速访问github的博文。

等待创建库完成。

连接

然后在博客根目录下右键，点击git Bash here打开git的窗口。
输入以下三条命令

然后输入 ssh-keygen -t rsa -C "你GitHub注册时用的邮箱" 获取秘钥和公钥。
打开github，个人头像-setting-SSH and GPG keys，然后新建一个ssh,title随便写。
打开 C:\用户\(你电脑的用户名)\.ssh\id_rsa.pub选择用记事本打开,ctrl+a全选，ctrl+c复制，黏贴到刚刚新建的ssh的key中点完成就好了。
输入 ssh -T git@github.com 问 Are you sure you want to continue connecting (yes/no)?时输入yes就可以了。

写&发布

新建文章

在git窗口中输入 hexo new post "文章标题" 新建文章，新建的文章在\source\_posts中以标题.md的形式呈现。使用markdown书写。你可以使用Typora或者VSCode预览和编辑

发布文章

在git窗口中输入 hexo d 上传到github库，第一次上传可能会要求你输入用户名和密码。
通过[你的用户名].github.io进行访问

注意

上传前需要用 hexo clean 清理缓存，然后使用 hexo g 渲染，最后使用 hexo d上传

如果成功，恭喜你，你完成了2/3了，如果你不想继续美化,仅仅想用来作为一个备份就全部完成了!

开始美化

Hexo的默认界面个人感觉很丑，于是我们需要使用主题包。
本人使用的是hexo-theme-melody这个主题,这里是他的官方页面。
这个主题对移动端比较友好，而且自动将css等资源使用cdn加速。

这里是官方快速入门，已经讲得很通俗且详细了。主题的_config.yml在\themes\melody里找，不需要复制到哪里..
需要注意的是:每次你更改了主题，需要按住ctrl+f5强制重新加载，否则css(页面)信息会直接从硬盘中读取，将不会有修改。

自定义域名

如果你觉得后缀是github.io太单调了，那么你可以自己购买一个域名，本人建议使用namesilo购买，一个com域名只需要60多，一个top域名只需要几块。而且不需要想国内服务商一样实名，你可以直接提供网名来验证。
推荐使用cloudflare进行解析，国内访问速度较快。
在库的setting中，找到 Custom domain 然后输入你要使用的域名。
然后在cloudflare添加CNAME解析，名称为githubpage的网址。
这样，你就可以通过自定义域名来访问你的博客了
